# Centrality Metrics Calculator

This module provides comprehensive centrality metrics calculation for inferred networks using the SNAP-py library. It processes networks generated by the netinf inference process and computes various topological properties and centrality measures.

## Features

### ✅ **SNAP-py Integration**

- **Pure SNAP Implementation**: All centrality calculations use SNAP-py functions
- **Memory Efficient**: Uses SNAP's efficient graph data structures

### ✅ **Centrality Metrics Calculated**

1. **Degree Centrality** - Using SNAP's node degree functions
2. **Betweenness Centrality** - Using `snap.GetBetweennessCentr()`
3. **Closeness Centrality** - Using `snap.GetClosenessCentr()`
4. **Eigenvector Centrality** - Using `snap.GetEigenVectorCentr()`
5. **PageRank** - Using `snap.GetPageRank()`
6. **Clustering Coefficient** - Using `snap.GetNodeClustCf()`
7. **Eccentricity** - Using `snap.GetNodeEcc()`

### ✅ **Flexible Processing Options**

- **Single Network**: Process one specific inferred network
- **Model-Specific**: Process all networks from one model (exponential, powerlaw, rayleigh)
- **Batch Processing**: Process all models automatically
- **Optional Plotting**: Generate distribution plots for each metric

### ✅ **Organized Output Structure**

```
data/
  centrality_metrics/
    exponential/
      centrality_metrics_exponential_000.csv
      centrality_metrics_exponential_001.csv
      ...
      (optional plot files)
    powerlaw/
      centrality_metrics_powerlaw_000.csv
      ...
    rayleigh/
      centrality_metrics_rayleigh_000.csv
      ...
```

## Usage Examples

### Method 1: Direct Function Calls

```python
from calculate_centrality_metrics import (
    calculate_centrality_for_network,
    calculate_centrality_for_all_models
)

# Process a single network with plots
success = calculate_centrality_for_network(
    "../data/inferred_networks/exponential/inferred-network-expo-000.txt",
    plot_enabled=True
)

# Process all models without plots
results = calculate_centrality_for_all_models(plot_enabled=False)
```

### Method 2: Command-Line Interface

```bash
# Process a single network with plots
python run_centrality_metrics.py --single-network ../data/inferred_networks/exponential/inferred-network-expo-000.txt --plots

# Process all networks from exponential model
python run_centrality_metrics.py --model exponential --plots

# Process all models (exponential, powerlaw, rayleigh)
python run_centrality_metrics.py --all-models

# Process with custom base path
python run_centrality_metrics.py --all-models --base-path /custom/path/to/networks --plots
```

## Main Functions

### `calculate_centrality_for_network(network_file, plot_enabled=False)`

Calculates centrality metrics for a specific inferred network file.

**Parameters:**

- `network_file` (str): Path to the inferred network file
- `plot_enabled` (bool): Whether to generate distribution plots

**Returns:**

- `bool`: Success status

### `calculate_centrality_for_all_models(plot_enabled=False)`

Processes all networks from all three models (exponential, powerlaw, rayleigh).

**Parameters:**

- `plot_enabled` (bool): Whether to generate distribution plots

**Returns:**

- `dict`: Results summary for each model

### `calculate_centrality_metrics_snap(G, plot_enabled, model_name, network_id, output_dir)`

Core function that calculates all centrality metrics using SNAP-py.

**Parameters:**

- `G`: SNAP graph object
- `plot_enabled` (bool): Whether to create plots
- `model_name` (str): Model name for output files
- `network_id` (str): Network identifier
- `output_dir` (str): Output directory

**Returns:**

- `dict`: Dictionary containing all calculated metrics

## Output Format

### CSV Files

Each processed network generates a CSV file with columns:

- `UserId`: User identifier
- `degree`: Node degree
- `betweenness`: Betweenness centrality
- `closeness`: Closeness centrality
- `eigenvector`: Eigenvector centrality
- `pagerank`: PageRank score
- `clustering`: Clustering coefficient
- `eccentricity`: Node eccentricity
- `community`: Community assignment

### Plot Files (Optional)

When `plot_enabled=True`, generates distribution plots for each metric:

- `Degree_distribution_modelname_networkid.png`
- `Betweenness_distribution_modelname_networkid.png`
- `Closeness_distribution_modelname_networkid.png`
- etc.

## Network File Format Expected

The inferred network files should follow this format:

```
# Nodes section
node_id,node_id
node_id,node_id
...

# Empty line separator

# Edges section
src_id,dst_id
src_id,dst_id
...
```

## Note on Model Names

The system expects network files following this naming convention:

- `inferred-network-expo-XXX.txt` → exponential model
- `inferred-network-power-XXX.txt` → powerlaw model
- `inferred-network-ray-XXX.txt` → rayleigh model

Where `XXX` is the network number (000, 001, 002, etc.).
